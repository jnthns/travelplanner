# Agent Skills

## Summary

The TravelPlanner repository ships with a collection of six agent skills that configure how AI coding assistants (Claude Code, Cursor, etc.) behave when working in this codebase. Two skills cover Gemini API usage — one for the core `generateContent` surface and one for the newer Interactions API. One skill overrides default shell assumptions to account for the Windows PowerShell host environment. Two skills (a SKILL.md and a README) define a pipeline for converting Stitch UI designs into React components. One skill defines best practices for authoring and editing Cursor rules in `.mdc` format. Together they form the agent-facing layer of project conventions, sitting alongside CLAUDE.md to ensure consistent, correct AI-generated code across tooling.

## Key Decisions

- **Two separate Gemini skills** rather than one: `gemini-api-dev` covers the `generateContent` / `generateText` surface used throughout `src/lib/gemini.ts` and the action modules; `gemini-interactions-api` is kept separate because its state model (`previous_interaction_id`) and SDK version requirements differ enough to warrant isolation.
- **PowerShell skill is always-on:** Because every shell command generated by an agent will run on the Windows host, the override must apply unconditionally rather than being opt-in.
- **Stitch pipeline uses system curl:** The `scripts/fetch-stitch.sh` fetch step deliberately bypasses AI tool fetch mechanisms to avoid token-limit issues when downloading large design payloads.
- **Cursor rules skill lives in `.cursor/skills/`** rather than `.agents/skills/` because it is Cursor IDE-specific and scoped to editing `.mdc` rule files, not general project coding.
- **Legacy SDK imports are banned:** Both Gemini skills reinforce that `@google/generative-ai` (old) is deprecated; only `@google/genai` is permitted, mirroring the CLAUDE.md directive.

|------------|--------|---------|--------|

| Importing from `@google/generative-ai` (legacy SDK) | Deprecated | Package is no longer maintained; structured output and newer model IDs are unavailable. All imports must use `@google/genai`. | [[../../../.agents/skills/gemini-api-dev/SKILL]] |
| Managing conversation history manually with generateContent | Superseded | The Interactions API's `previous_interaction_id` eliminates the need to pass full history arrays, reducing payload size and simplifying stateful agent flows. | [[../../../.agents/skills/gemini-interactions-api/SKILL]] |
| Fetching Stitch design assets via AI fetch tools | Failed | Large design payloads exceed token limits; `scripts/fetch-stitch.sh` using system-level curl is the reliable alternative. | [[../../../.agents/skills/react-components/SKILL]] |
| AST validation of generated React components via `@swc/core` | Active | Catches structural errors (missing exports, invalid JSX) before the 20-point architecture audit runs, reducing review cycles. | [[../../../.agents/skills/react-components/README]] |

## Gotchas & Known Issues

- **PowerShell `&&` silently breaks multi-step commands.** Any AI-generated shell sequence that chains commands with `&&` will fail on this Windows host without a clear error message in some contexts. Always use `;` or issue commands as separate sequential calls. The `powershell-syntax` skill exists specifically to prevent this, but agents that do not load it will regress.
- **Legacy Gemini SDK imports (`@google/generative-ai`).** Auto-complete and older documentation still surface this package. It is incompatible with current model IDs (`gemini-3-*`) and lacks structured output support. Any import from `@google/generative-ai` must be replaced with `@google/genai`.
- **Stitch asset fetch must use system curl, not AI tool fetching.** The `react-components` skill's download step (step 3) requires `scripts/fetch-stitch.sh` because AI tool fetch mechanisms cannot handle the payload sizes returned by Stitch design exports. Skipping this step and attempting an in-agent fetch will produce truncated or errored downloads.
- **Interactions API SDK version floor.** The `gemini-interactions-api` skill requires `@google/genai >= 1.33.0` (JS/TS) or `google-genai >= 1.55.0` (Python). Older installed versions will not expose the `previous_interaction_id` field and will fail silently with standard `generateContent` behavior.
- **Cursor rules temporal references are forbidden.** The `cursor-rules` skill explicitly lists "temporal references" as an anti-pattern in `.mdc` files. Do not write rules that reference dates, version numbers that will change, or phrases like "currently" or "as of."
- **`gemini-3-pro-image-preview` is the only model for image input.** Other `gemini-3-*` models do not accept image modalities. Attempting multimodal prompts on `gemini-3-flash-preview` or `gemini-3-pro-preview` will be rejected by the Cloudflare Worker's allowed-models list.

## Open Questions

- Should `gemini-api-dev` and `gemini-interactions-api` be merged into a single skill with subsections, now that the Interactions API has stabilized, or does the SDK version difference justify keeping them separate?
- The `react-components` skill references `scripts/fetch-stitch.sh` — does this script exist in the repository, and is it committed or gitignored?
- Is the `powershell-syntax` skill applied globally to all agents, or does it depend on each tool loading it? If Cursor does not load `.agents/skills/`, Windows shell failures will recur in that environment.
- The `cursor-rules` skill documents four rule types but does not specify which types are in use in `.cursor/rules/` — should an inventory of existing `.mdc` files and their types be maintained here?
- Are there plans to add skills for Firebase Firestore conventions (e.g., enforcing `stripUndefined`) or date-fns usage to give agents the same guardrails that CLAUDE.md gives human developers?
